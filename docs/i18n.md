# Internationalisation (i18n)

## Supported languages

| Code | Language | Fallback |
|---|---|---|
| `da` | Danish | — (primary fallback) |
| `en` | English | — |

The fallback language is **Danish** (`fallbackLng: 'da'`). If a key is missing in the detected language, i18next falls back to Danish rather than English. This reflects that the project's primary audience is Danish-speaking.

## Setup

Configuration lives in `frontend/src/i18n/config.ts`. It initialises i18next with:

- `i18next-browser-languagedetector` — reads from `localStorage` key `bridge-lang` first, then from `navigator.language`.
- `react-i18next` — provides the `useTranslation` hook and `<Trans>` component.
- All locale files are imported statically (no lazy loading).

The config is imported once in `frontend/src/main.tsx` as a side-effect (`import './i18n/config'`).

## File structure

```
frontend/src/i18n/
  config.ts                 -- i18next initialisation
  locales/
    en/
      common.json           -- nav, sort, tags, actions, gallery, profile, language labels
      auth.json             -- login / register form strings
      editor.json           -- system editor, convention editor, bid detail, share dialog
    da/
      common.json
      auth.json
      editor.json
```

Three namespaces mirror the three JSON files: `common`, `auth`, `editor`.

## Using translations in components

Components call `useTranslation` with one or more namespace names:

```tsx
const { t } = useTranslation('editor');
const { t } = useTranslation(['editor', 'common']);
```

When multiple namespaces are listed, the first one is the default — bare keys resolve against it. Cross-namespace keys are prefixed with the namespace:

```tsx
t('common:action.cancel')   // explicit namespace prefix
t('saveIndicator.saving')   // resolves against the default namespace
```

Interpolation uses double-brace syntax: `t('tag.sharedBy', { username, permission })` matches `"Shared by {{username}} · {{permission}}"` in the locale file.

## Adding a new translation key

1. Add the key to `frontend/src/i18n/locales/en/<namespace>.json`.
2. Add the same key to `frontend/src/i18n/locales/da/<namespace>.json`.
3. Use it in the component: `t('<key>')` (or `t('<namespace>:<key>')` for cross-namespace).

If a key exists in one locale but not the other, i18next falls back to Danish. Missing keys in both locales render the key path as a string — this is visible in the UI and should be treated as a build error.

## Language switcher

The `bridge-lang` key in `localStorage` controls the stored preference. Components that render a language toggle write to this key and call `i18n.changeLanguage(code)`. The `common.json` files carry the display names: `language.da = "Dansk"` and `language.en = "English"`.
